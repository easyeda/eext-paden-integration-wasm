# PADEN Симуляция

Расширение JLCEDA & EasyEDA Pro — извлечение данных PCB и выполнение FEM-анализа PDN (сети распределения питания)

> **Версия**: 1.0.7 | **Категория**: PCB | **Ключевые слова**: PDN, Power Analysis, Simulation, 仿真, PI

## Возможности

- Извлечение трасс, переходных отверстий, контактных площадок и медных заливок из EasyEDA
- Пользовательская конфигурация шин питания (источники напряжения, токовые нагрузки)
- Предварительная сетка на клиенте (триангуляция earcut на TypeScript)
- Локальный Python-бэкенд для FEM-решения
- WebGL-визуализация распределения напряжения и тепловой карты плотности мощности

## Быстрый старт

### 1. Установка зависимостей frontend

```shell
npm install
```

### 2. Компиляция расширения

```shell
npm run compile
```

Сборка для релиза:

```shell
npm run build
```

### 3. Запуск Python-бэкенда

```shell
cd paden-service
start-paden-windows.bat
```

`start-paden-windows.bat` автоматически:
- Загружает последние `solver.py` и `problem.py` с GitHub
- Устанавливает Python-зависимости (numpy, scipy, shapely, fastapi, uvicorn, matplotlib)
- Проверяет синтаксис
- Запускает сервер на `localhost:5000`

### 4. Установка в EasyEDA

1. Откройте JLCEDA Pro, войдите в редактор PCB
2. Установите пакет расширения. При первом запуске появится подсказка для запуска бэкенд-сервиса — следуйте инструкциям

![Запуск сервиса](./images/img-5.png)

3. Выберите **PDN анализ → Запустить PDN анализ...** в меню

## Структура проекта

```
├── src/                    # TypeScript frontend
│   ├── index.ts            # Главная точка входа, организация анализа
│   ├── extract.ts          # Извлечение данных PCB (трассы, VIA, площадки, медь)
│   ├── convert.ts          # Конвертация данных + предварительная сетка + сериализация
│   ├── mesh.ts             # Клиентская триангуляция (earcut, полурёберная структура)
│   ├── api.ts              # HTTP-коммуникация (с Python-бэкендом)
│   ├── display.ts          # Отображение результатов (IFrame + Storage + MessageBus)
│   └── types.ts            # Определения типов
├── ui/
│   ├── config.html         # UI конфигурации шин питания
│   ├── results.html        # WebGL визуализация результатов
│   └── results.tpl.html    # Шаблон сборки results.html
├── paden-service/          # Python-бэкенд
│   ├── main.py             # FastAPI сервер (десериализация, решение, визуализация)
│   ├── solver.py           # FEM-решатель (с GitHub)
│   ├── problem.py          # Определение задачи (с GitHub)
│   ├── mesh_pure.py        # Структуры данных сетки (полурёбра, дифференциальные формы)
│   ├── standby/            # Резервные копии solver.py + problem.py
│   └── start-paden-windows.bat           # Скрипт сборки и запуска в один клик
├── config/                 # Конфигурация сборки
│   ├── esbuild.common.ts
│   └── esbuild.prod.ts
└── extension.json          # Манифест расширения
```

## Рабочий процесс

1. **Извлечение данных** — Извлечение трасс, переходных отверстий, площадок и медных областей из текущего PCB
2. **Конфигурация анализа** — Выбор цепей питания, настройка источников напряжения и токовых нагрузок
3. **Предварительная сетка** — Триангуляция медных областей алгоритмом earcut на TypeScript
4. **FEM-решение** — Python-бэкенд получает предсеточные данные, строит матрицу Лапласа и решает распределение напряжения
5. **Визуализация** — Тепловая карта напряжения WebGL с переключением слоёв, отображением рёбер сетки и маркерами VIA

## Технологии

**Frontend**: TypeScript, esbuild, WebGL, earcut

**Backend**: Python, FastAPI, numpy, scipy, shapely, matplotlib

**Зависимости**: `@jlceda/pro-api-types`, `earcut`
